# CardLift Demo Deck

## Purpose

The CardLift Demo Deck exists to help first-time users become
comfortable with CardLift before importing their own PDFs.

Its goals are to build confidence that:

-   CardLift works as promised.
-   The user can successfully use the application.
-   The user finishes the demo wanting to try **their own** deck.

The Demo Deck is **not** intended to be a game, a marketing asset, or a
showcase of artwork. It is an onboarding experience.

------------------------------------------------------------------------

## User Experience Goals

A first-time user should feel:

-   Welcomed
-   Invited
-   Calm
-   Curious
-   Confident

By the end of the demo, the desired reaction is:

> "I can't wait to see what my own deck looks like."

------------------------------------------------------------------------

## Design Principles

-   Standard poker card size (2.5 × 3.5 in)
-   Clean, modern layout
-   White background
-   Thin square outline on both front and back
-   Outline slightly darker than other design elements to aid
    calibration
-   Soft purple accent palette
-   Simple outline icons from a single icon family
-   Large title
-   Short subtitle
-   Generous whitespace
-   Shared card back
-   Consistent layout across all cards

------------------------------------------------------------------------

## Content Principles

The Demo Deck is intentionally:

-   Not a game
-   Not a story
-   Not a tutorial
-   Not heavily branded

Each card should stand on its own.

The only intentional transition is the final card:

**Your Turn**

which gently invites the user to import their own PDF.

------------------------------------------------------------------------

## First Card

**Light the Way**

*Begin here.*

This establishes the tone without explaining the application.

------------------------------------------------------------------------

## Final Card

**Your Turn**

This serves as a quiet handoff from the Demo Deck to the user's own
project.

------------------------------------------------------------------------

## Success Criteria

The Demo Deck succeeds when users remember the experience---not the
cards.

If the demo causes users to immediately load one of their own PDFs, it
has achieved its purpose.

------------------------------------------------------------------------

## Open Question

Should the PDF itself contain any introductory material, or should all
onboarding remain within the CardLift application while the Demo Deck
behaves like a normal card deck?

This question remains intentionally open for product review.
